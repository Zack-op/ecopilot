## Current Session: Implementation Suggestions Feature

### What Was Added
For every contributor action, the LLM now generates 2-3 practical implementation suggestions.

**Key Constraint:** LLM generates SUGGESTIONS ONLY. Actions remain generated by Python.

### Architecture
```
Insight Generator (Python)
├─ Generates: actions for each contributor
└─ Called by: LLM Coach

LLM Coach
├─ Task 1: Generate coach message (existing)
├─ Task 2: Generate implementation suggestions (NEW)
│   └─ Input: Pre-generated action from Python
│   └─ Output: 2-3 practical ways to achieve that action
└─ Constraint: Does NOT generate actions
```

### Key Functions Added

#### `LLMInsightGenerator.generate_implementation_suggestions(insight_result)`
- Takes insights with contributors
- Returns dict mapping contributor index to list of suggestions
- Generates 2-3 practical suggestions per action

#### `build_suggestion_prompt(action)`
- Creates LLM prompt with strict constraints
- Prevents LLM from generating or modifying actions
- Focuses only on "ways to achieve" the action

#### `build_local_action_suggestions(action)`
- Deterministic fallback suggestions
- Category-specific (video, screen, social, charging, music)
- Returns 3 suggestions always (no randomness)

#### `_parse_suggestions_from_response(response)`
- Extracts bullet-point suggestions from LLM output
- Limits to 1 sentence per suggestion
- Returns max 3 suggestions

## Key Design Decisions

1. **Deterministic Fallback**: Suggestions work perfectly without LLM
2. **Category-Specific**: Different action types get relevant suggestions
3. **Limited Scope**: Max 3 suggestions (not overwhelming)
4. **Action Integrity**: Python generates, LLM only suggests how to achieve
5. **Simple Constraints**: Clear LLM instructions prevent feature drift
